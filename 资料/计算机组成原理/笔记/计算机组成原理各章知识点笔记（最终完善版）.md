# 计算机组成原理各章知识点笔记（最终完善版）

## 第一章 计算机系统概论

### 1.1计算机系统简介

1. **计算机系统组成**：  

    - 硬件系统：CPU（运算器+控制器）、存储器（主存/辅存）、I/O设备、系统总线。  
    - 软件系统：系统软件（操作系统、编译程序、链接程序、汇编程序）、应用软件（文本处理、游戏等）。  
2. **语言处理程序流程**：  
    - 高级语言→预处理→编译（生成汇编语言）→汇编（生成目标文件）→链接（生成可执行文件）。  

    - 执行方式：机器语言直接执行；汇编语言需汇编程序转换；高级语言可编译（一次性转换）或解释（逐句执行，不生成目标文件）。  
3. **软硬件关系**：  
    - 软硬件在**逻辑功能上等效**（同一功能可由硬件或软件实现，如硬件加速替代软件功能）可替代；
    - 硬件**执行速度通常高**于软件。

### 1.2计算机基本组成

1. **冯·诺依曼体系结构核心**：  

    - 工作方式：控制流驱动方式

    - 存储程序原理：指令和数据以二进制存于主存，按地址访问并顺序执行。  

    - 五大组成部分：运算器、控制器、存储器、输入设备、输出设备。  

    - 二进制编码：运算规则简单、易实现物理器件、便于逻辑门电路运算。  

    - **指令与数据的区分**：CPU通过**指令周期的不同阶段**区分（取指周期读指令，执行周期读数据），而非存储单元或形式。  

2. **CPU核心部件**（运算器、控制器、寄存器）：  

    运算器必需：**ALU**算术逻辑单元 + 累加器**ACC** + 数据暂存器**TR** + 数据缓冲寄存器**DR** + 状态寄存器**SR/FR**；

    控制器必需：**PC** 程序计数器（地址寄存）+ **IR**指令寄存器 + 指令译码器**ID** + 控制单元（**IR**） +程序状态字寄存器 **PSW** + 时序产生器。

    书：

    - 运算器（3个寄存器+ALU算术逻辑单元）：ALU、累加器ACC、MQ乘商寄存器、X操作数寄存器。  

    - 控制器（3）：CU控制单元、PC（程序计数器，存放下一条指令地址）、IR（指令寄存器，存放当前指令。

### 1.3 计算机硬件的主要技术指标

1. **性能指标**（运算速度）：  

    - 主频：CPU时钟频率；
    - CPI：执行一条指令平均时钟数（时钟周期）；
    - MIPS：每秒百万条指令（通用指标）；
    - MFLOPS：每秒浮点运算次数（浮点数专用，**标志系统性能的主要指标**）。  

    **关键公式：**

    - 程序执行时间 = 总时钟周期数 × 时钟周期（时钟周期 = 1 / 主频）= IC× 平均 CPI ÷ f；

      ​					总时钟周期数 = 指令数（IC）× 平均 CPI；

      ​					平均 CPI = Σ（各类指令所占比例 × 对应 CPI）；

    - MIPS = 主频（Hz）÷（平均 CPI × 10⁶(与M相关)）。

    - （同上）用户 CPU 时间：程序占用 CPU 的总时间（= 指令数 IC× 平均 CPI÷f）

    **典型例题：**

    - 例 1（2017 联考）：M1 主频 1.5GHz、CPI=2；M2 主频 1.2GHz、CPI=1（ISA 相同，IC 相同）→ 时间比 T₁/T₂=(2×1.2)/(1×1.5)=2.4/1.5=1.6。
    - 例 2（2023 联考）：主频 1.5GHz，IC=5×10⁵，平均 CPI=1.2 → ①指令执行速度（GIPS）=1.5×10⁹÷(1.2×10⁹)=1.25；②CPU 时间 =(5×10⁵×1.2)÷(1.5×10⁹)=6×10⁵÷1.5×10⁹=4×10⁻⁴秒 = 0.4ms。
    - 例 3（原考点）：主频 1.2GHz，指令 A（50%，CPI=2）、B（20%，CPI=3）、C（10%，CPI=4）、D（20%，CPI=5）→ 平均 CPI=3，MIPS=（1.2×10⁹）÷（3×10⁶）=400。

2. **机器字长**：CPU一次能处理数据的位数。（32位**微机**）

3. **主存容量 = 2^储存单元个数（MAR）×储存字长（MDR）**

习题：

1. 程序执行时间（ 主频（Hz）、指令数（IC）、平均 CPI）优化措施

   （3 类均有效）：

   - I. 提高 CPU 时钟频率：缩短单个时钟周期（总时钟周期数不变时，直接减少执行时间）；
   - II. 优化数据通路结构：降低 CPI（减少指令执行的时钟周期数）；
   - III. 编译优化：减少指令数或优化执行顺序（降低总时钟周期数）。

2. **CPU 速度提升与总时间的关系**

   - 程序总时间 = CPU 时间 + I/O 时间（I/O 时间不受 CPU 速度影响，保持不变）；
   - CPU 速度提高 n% → 新 CPU 时间 = 原 CPU 时间 ÷（1+n%）；
   - 典型例题：原总时间 100 秒（CPU 时间 90 秒，I/O 时间 10 秒），CPU 提速 50% → 新 CPU 时间 = 60 秒，总时间 = 60+10=70 秒。

3. 指令数与 CPI 变化对执行时间的影响

   - 核心逻辑：时钟周期（T）不变时，新执行时间 = 原执行时间 ×（指令数缩减比例）×（CPI 提升比例）；
   - 典型例题：原执行时间 20 秒，指令数减至 70%，CPI 增至 1.2 倍 → 新时间 = 20×0.7×1.2=16.8 秒。

4. 地址空间计算

   - 地址空间 = 2^ 地址位数（地址空间是可寻址的存储单元总数，与机器字长无关）；
   - 典型例题：16 位地址 → 地址空间 = 2¹⁶=65536 个，答案选 C。

5. 典型例题（综合应用题）：

   题目：某计算机仅 Load/Store 访存，指令占比及 CPI 如下：算术逻辑指令 43%（CPI=1）、Load21%（CPI=2）、Store12%（CPI=2）、转移 24%（CPI=2）。

   （1）基础平均 CPI 计算：

   平均 CPI=（43%×1）+（21%×2）+（12%×2）+（24%×2）=0.43 + 0.42 + 0.24 + 0.48=1.57。

   （2）指令集优化后平均 CPI 计算（新增寄存器 - 存储器型算术逻辑指令）：

   ① 假设原程序总指令数为 M，算术逻辑指令数 = 0.43M；

   ② 优化逻辑：25% 的算术逻辑指令（0.43M×25%=0.1075M）需额外 Load，现用新指令替代（无需额外 Load）→ 减少 Load 指令数 = 0.1075M，新增新指令数 = 0.1075M，原算术逻辑指令数变为 0.43M×75%=0.3225M；

   ③ 新总指令数 = 原总指令数 - 减少的 Load 指令数 + 新增指令数 = M - 0.1075M + 0.1075M = M（总指令数不变，仅类型占比变化）；

   ④ 各类指令新占比：

   - 原算术逻辑指令：0.3225M/M=32.25%（CPI=1）；
   - 新算术逻辑指令：0.1075M/M=10.75%（CPI=2）；
   - Load 指令：(0.21M - 0.1075M)/M=10.25%（CPI=2）；
   - Store 指令：12%（CPI=2）；
   - 转移指令：24%（CPI=3，优化后变化）；

   ⑤ 新平均 CPI=（32.25%×1）+（10.75%×2）+（10.25%×2）+（12%×2）+（24%×3）=0.3225 + 0.215 + 0.205 + 0.24 + 0.72=1.7025≈1.70。

## 第三章 系统总线

### 3.1总线的基本概念

### 3.2总线的分类

1. **总线分类与功能**：  
    - 按连接部件：片内总线（芯片内部）、系统总线（CPU、主存、I/O设备各部件的数据传输线）、通信总线（计算机之间或计算机与其他系统之间的通信）
    - 按层次：系统总线（连接CPU、主存、I/O接口）、局部总线（PCI/AGP，连接高速外设，**作用是节省系统总带宽**）、通信总线（**USB：通用串行总线**，支持即插即用、级联）。  
    - 按传输方式：串行总线（USB，一次传1位）、并行总线（ISA/PCI，一次传多位）。  
    - **系统总线 按传输信息**：地址总线（单向，CPU→主存 / I/O，传主存/I/O地址）、数据总线（双向，传指令/操作数/中断类型码，传输主存 / I/O→CPU 的信息）、控制总线（双向，传握手信号/读/写控制）。  

### 3.3总线特性及性能指标

1. **总线性能指标与计算**：  

    - 带宽（数据传输速率）：公式1=（总线位宽（位 /8 转字节）× **时钟频率**）/（时钟周期数量）

    - **乘频率**或**除周期**
        - 例子1：32位总线、500MHz时钟、5个时钟周期传1字→带宽=（4字节×500MHz）/5=400MB/s；  
        
        - 例子2：5MHz时钟、4个时钟周期为总线周期→总线周期=4×(1/5MHz)=0.8×10⁻⁶s，带宽=4字节÷0.8×10⁻⁶s=5MB/s。  
        
    - 位宽：一次传输的二进制位数（与CPU字长相关）；时钟频率：总线工作时钟。  

2. **常见总线标准**：  

    | 总线名称                | 连接层次/功能定位分类                                        | 数据传输格式分类                           | 专用性分类                 |
    | ----------------------- | ------------------------------------------------------------ | ------------------------------------------ | -------------------------- |
    | **ISA总线**             | 系统总线                                                     | 并行总线（8位→16位）                       | 通用系统总线               |
    | **EISA总线**            | 系统总线                                                     | 并行总线（32位）                           | 通用系统总线               |
    | **PCI总线**             | 1. 系统总线（替代ISA，连接CPU/主存/I/O桥） 2. 局部总线（连接高速外设） | 并行总线（32位→64位）                      | 通用总线                   |
    | **PCI-Express（PCIe）** | 1. 系统总线（根复合体连接CPU/主存） 2. 局部总线（连接高速外设） | 串行总线（点对点通道）                     | 通用总线                   |
    | **VESA（VL-BUS）**      | 局部总线                                                     | 并行总线（32位）                           | 通用局部总线               |
    | **AGP总线**             | 局部总线（显卡专用）                                         | 并行总线（32位）                           | 专用总线（仅显卡）         |
    | **USB总线**             | 通信总线（外部总线）                                         | 串行总线                                   | 通用通信总线               |
    | **SCSI总线**            | 1. 通信总线（外部总线，连接存储设备） 2. 局部总线（连接主机内存储控制器） | 1. 早期：并行总线 2. 后期（SAS）：串行总线 | 通用通信总线（主用于存储） |
    | **RS-232C总线**         | 通信总线（外部总线）                                         | 串行总线                                   | 通用通信总线               |

    - 系统总线：ISA、EISA、PCI、PCI-Express；  

    - 局部总线：PCI、PCI-Express、VESA（VL-BUS）、AGP、SCSI

    - 通信总线：USB、SCSI、RS-232C

    - 非总线标准：CRT（显示器）、CPI（性能指标）、RAM（存储器）、MIPS（性能指标）。

3. **总线关键技术与性能**：

    - **信号线复用技术**：同一信号线分时传输不同信号（如地址 / 数据线复用），**可减少信号线数量**，但不直接提高传输率。
    - **突发传输方式**：连续传输多个数据块时，仅传输一次地址，后续直接传数据，**减少地址传输开销，提高总线数据传输率**。
    - **分离事务通信方式**：主设备发送请求后可执行其他操作，待从设备准备好数据后再回传，**提高总线利用率**。
    - **同步总线传输率提升方法**：增加总线带宽（位宽）、提高总线工作频率、支持突发传输；地址 / 数据线复用不提升传输率（分时传输增加延迟）。

### 3.4总线结构

1. **组成**：分为单总线结构和多总线结构两种。

2. **单总线结构：**

   - 结构：将CPU、主存、I/O设备挂在一组总线上，相互之间直接交换信息。
   - 适用：小型/微型计算机

3. **多总线结构：**

   - 结构：将I/O设备分离出来，通过通道与主存总线连接。（双）

     ​			再用速率分类I/O设备，连到不同的通道上（多）

   - 适用：大/中型计算机

### 3.5总线控制

**总线仲裁（主设备控制权分配）**：

- 核心规则：总线上**同一时刻只能有一个主设备控制传输操作**（主设备是发起传输的设备）。
- “总线忙” 信号：由**获得总线控制权的主设备**建立，表示总线正在被占用。

1. **集中式总线仲裁**：  

    - 链式查询：控制线少，设备优先级固定，对故障敏感（前级故障影响后级）。  

    - 计数器定时查询：控制线中等，优先级可循环（从上次终止点计数，**机会均等**）。  

    - 独立请求：控制线多（**2n+1条**），响应速度最快，优先级可编程（**可编程为机会均等**）。  

    

2. **总线通信方式**：  

    - 同步通信：统一时钟控制，适用于速度相近设备，无握手信号，传输率高。**时钟信号**由**总线控制器 / CPU集中提供**
        - **数据传输率（单位Bps 字节/秒 也存在位/秒）**
        - **每个数据位占用的时间：1 / 波特率**

    - 异步通信：无统一时钟，靠“请求-应答”握手信号，适用于速度差异大设备，分三种：  

        - 不互锁（最快）：发送方发信号后直接结束；  

        - 半互锁：发送方 等待接收方应答后结束；  

        - 全互锁（最慢，最可靠）：发送方等应答，接收方等发送方确认。  

        - **传输率（位 / 秒）= 每秒传输字符数 × 数据总位数**（单位时间内传输二进制数据位数）

        - **波特率（波特，全部数据位数）、比特率（有效数据位数）（单位bps 位/秒）**
    - 半同步通信：同步+异步（"等待"响应信号），握手信号的采样由同步时钟控制，适用于速度不高且相近的设备
    - 分离式通信：适用于大型计算机

习题补充：

1. **异步串行的标准传输规则：**
   - **数据位的传输顺序是 “先传最低位（D0），后传最高位（D7）”**
   - 书写字节的二进制数时，习惯以 “**最高位在前、最低位在后**” 的顺序表示



## 第四章 存储器

### 4.1 存储器基础

1. **存储器分类**：  

    | 分类维度        | 具体类型                | 核心特性                             | 典型示例                                      |
    | --------------- | ----------------------- | ------------------------------------ | --------------------------------------------- |
    | 按存取方式      | 随机存取（RAM/ROM）     | 直接按地址访问，存取速度与位置无关   | SRAM、DRAM、EPROM                             |
    |                 | 直接存取（DAM）         | 先定位区域，再顺序访问子单元         | 硬盘、磁盘                                    |
    |                 | 顺序存取（SAM）         | 按数据顺序访问，速度最慢             | 磁带、CD-ROM                                  |
    |                 | 只读（ROM）             |                                      |                                               |
    | 按易失性        | 易失性存储器            | 断电后数据丢失                       | SRAM（构成Cache）、DRAM（主存）               |
    |                 | 非易失性存储器          | 断电后数据保留                       | ROM、EPROM、Flash、CD-ROM、硬盘（磁盘）、光盘 |
    | 按存储位置/功能 | 内存储器（主存）        | CPU可直接访问，速度快、容量小        | RAM+ROM                                       |
    |                 | 外存储器（辅存）        | CPU需通过I/O接口访问，容量大、速度慢 | 硬盘、U盘、CD-ROM                             |
    |                 | 高速缓冲存储器（Cache） | 介于CPU与主存之间，缓解速度差异      | SRAM构成                                      |
    
    - 存储介质：半导体存储器、磁表面存储器、磁芯存储器、光盘存储器
    - 操作系统保存在硬盘上，内存储器应用RAM和ROM
    
    2.**存储器层次结构**：  
    
    - 目的：平衡 “速度、容量、成本” 三者矛盾，实现 “高速、大容量、低成本” 综合需求。
    
    - 层次顺序（从快到慢、成本从高到低）：寄存器（CPU 内部）→ Cache（缓存）→ 主存  → 辅存（磁盘、磁带）
    
    
    - CPU 访问规则：优先访问高层存储，未命中时向下层查找，依赖局部性原理提升效率。 
    
    - CPU访问顺序：先Cache→主存→辅存（CPU不能直接访问辅存，需通过I/O接口调入主存）。
    - 缓存-主存：解决CPU和主存速度不匹配问题
    - 主存-辅存：解决存储系统容量问题

#### 4.2.5 习题

##### 一、存储器扩展（位扩展与字扩展）

1. **芯片数量计算**

   - 总容量 = 单芯片容量 × 芯片数量
   - 位扩展：需满足总位数 = 单芯片位数 × 并行芯片数（如用 4K×4 位芯片组成 4K×8 位，需 2 片并行）
   - 字扩展：需满足总地址范围 = 单芯片地址范围 × 扩展芯片数（如用 2K×4 位芯片组成 8K×8 位，需 8 片：4 组 ×2 片并行）

   *例*：用 4K×4 位芯片组成 64KB（8位） 存储器（含 4KB ROM）

   - ROM 区：8位/4位= 2 片
   - RAM 区：60K/4K × 8位/4位 = 30 片

2. **地址线分配**

   - 片内地址线：由单芯片容量决定（如 2K=2¹¹，需 11 位片内地址）
   - 片选地址线：剩余高位地址用于选择芯片 / 芯片组（如 16KB 存储器用 4K×4 位芯片，片内 12 位，片选 2 位）

   *例*：地址 0B1FH（0000 1011 0001 1111B）对应 8K×8 位存储器（2K×4 位芯片组成）

   - 高 3 位（001）选择芯片组，地址范围 0800H（001 000 0000 0000）-0FFFH

##### 二、地址空间与 MAR 位数

1. **MAR 位数**：由主存地址空间大小决定，与实际安装容量无关
   - 公式：若地址空间为 2ⁿ字节，则 MAR 至少 n 位
   - *例*：64MB（2²⁶B）地址空间，MAR 需 26 位
2. **地址范围计算**
   - 地址区间单元数 = 末地址 - 首地址 + 1
   - *例*：地址 4000H-43FFH，单元数 = 43FFH-4000H+1=400H=1024 个

### 4.3 高速缓冲存储器（Cache）

1. **命中率**

   - 命中率 = 命中次数 ÷ 总访存次数 × 100%
   - *例*：1000 次访存，50 次缺失，命中率 = 950/1000=95%

2. **映射方式（组相联）**

   - 组号 = 主存块号 mod 组数
   - *例*：Cache 共 16 块（二路组相联，8 组），主存块大小 32 字节，129 号单元所在主存块号 = 4，组号 = 4 mod 8=4

3. **数据交换**

   - 失效时：主存→CPU 以字为单位（立即使用），同时主存→Cache 以块为单位（空间局部性）

4. **写策略**

   - 全写法（Write Through）：同时更新 Cache 和主存
   - 写回法（Write Back）：仅更新 Cache，替换时写回主存

5. **替换算法**

   - LRU（近期最少使用法）：基于局部性原理，优先替换最近少用的块

6. **局部性原理**

   - 时间局部性：近期访问的数据可能再次访问（如循环变量）
   - 空间局部性：连续地址的数据可能被顺序访问（如数组）
   - *例*：数组访问同时具有时间和空间局部性

7. **主要公式**：

   - 芯片数量 = （总容量 × 总位数）÷（单芯片容量 × 单芯片位数）

   - 主存块号 = 单元地址 ÷ 块大小（取整数）

   - 命中率 = （总访存次数 - 缺失次数）÷ 总访存次数

   - 地址线位数 = log₂（总单元数）

## 第五章 输入输出系统

### 一、中断系统（核心考点）

#### 1. 中断的分类

| 分类维度       | 类型                | 触发源/特点                        | 典型例子               |
| -------------- | ------------------- | ---------------------------------- | ---------------------- |
| 按触发源划分   | 外部中断（硬中断）  | 外部设备/硬件触发                  | 键盘输入、打印机缺纸   |
|                | 内部中断（异常）    | CPU内部事件/程序错误触发           | 除数为0、访存缺页      |
| 按响应权限划分 | 可屏蔽中断          | 需CPU“开中断”+指令执行结束才响应   | 普通外设中断           |
|                | 不可屏蔽中断（NMI） | 不受屏蔽位影响，指令执行结束后响应 | 电源故障、系统紧急中断 |

中断屏蔽字主要用于控制哪些中断可以被响应，哪些中断被屏蔽。

1. 单重中断：在处理一个中断时，会关闭中断（关中断），以避免被新的中断打断，处理完毕后再打开。
2. 多重中断：允许中断嵌套。为了实现这一点，系统在保护现场后（即进入中断服务程序初期）会重新打开中断，使得更高优先级的中断可以打断当前的中断服务-3。因此，在整个中断处理期间，CPU并非一直处于关中断状态
3. 中断响应的完整过程：
   1. 中断请求：外部设备通过中断请求线（IRQ）发出信号。
   2. 中断响应：CPU在当前指令执行结束时检查并响应中断。
   3. 保护现场：CPU自动将程序计数器（PC）等关键状态压入堆栈。
   4. 执行服务程序：转向中断服务程序入口。在多重中断系统中，此时会开启中断，允许嵌套。
   5. 恢复现场与返回：执行IRET等指令，弹出保存的状态，返回原程序继续执行

#### 2. 中断响应的核心流程

中断响应分为**硬件自动执行（中断隐指令）**和**软件执行（中断服务程序）**两部分：

- **中断隐指令（硬件操作）**（必考）：

  1. 关中断（防止响应过程被干扰）；

  2. 保护断点（保存`PC`（程序计数器）+ `PSW`（程序状态字））；

  3. 发中断响应信号（通知中断源）；

  4. 形成中断向量地址，送入`PC`（跳转至中断服务程序）。

- **中断服务程序（软件操作）**：

保存通用寄存器 → 执行I/O操作 → 恢复通用寄存器 → 开中断 → 中断返回（恢复`PC+PSW`）。

#### 3. 中断相关计算（高频题型）

##### （1）中断时间占比计算

公式：`时间占比 =（单次中断处理时间 ÷ 中断请求周期）× 100%`

##### （2）中断时钟周期计算

步骤：

1. 算**单次中断开销**：`单次开销 = 中断响应时钟数 + 中断服务程序指令数 × CPI`；

2. 算**总开销**：`总时钟周期 = 单次开销 × 中断次数`（中断次数由设备传输率/缓冲字节数决定）。

### 二、DMA方式（直接内存访问）

##### 1. 核心特性

- **传输路径**：内存 → 数据总线 → DMA控制器 → 外设（**全程不经过CPU**）；

- **总线控制权**：传输期间由DMA控制器接管，CPU释放总线；

- **对CPU的影响**：不破坏`PC`/寄存器内容，CPU可执行非访存指令。

##### 2. DMA的执行主体

DMA方式的执行主体是**DMA控制器（硬件）**，无需CPU执行任何传送程序（区别于查询/中断方式）。

### 三、I/O控制方式对比（必考选择题）

| 控制方式 | 执行主体            | 并行性                          | 适用设备           |
| -------- | ------------------- | ------------------------------- | ------------------ |
| 查询方式 | CPU执行查询程序     | 无并行性（CPU与外设串行）       | 低速设备（键盘）   |
| 中断方式 | CPU执行中断服务程序 | CPU与外设并行，数据传送串行     | 中速设备（打印机） |
| DMA方式  | DMA控制器（硬件）   | 全并行（CPU/外设、传送/主程序） | 高速设备（磁盘）   |

### 四、I/O端口与编址

##### 1. 端口编址方式

| 编址方式 | 地址空间   | 访问指令                     |
| -------- | ---------- | ---------------------------- |
| 独立编址 | 与主存分离 | 专用I/O指令（如`IN/OUT`）    |
| 统一编址 | 与主存共享 | 访存指令（视端口为主存单元） |

##### 2. I/O指令的数据传送对象

I/O指令实现**CPU通用寄存器 ↔ I/O端口**之间的数据传送，I/O端口再与外设交互数据。

### 五、异步串行通信计算

- **单字符总位数**：`起始位（1位） + 数据位 + 校验位 + 停止位`；

- **每秒最大字符数**：`1s ÷ 单字符处理耗时`。

### 六、核心易错点总结

1. DMA方式无需CPU执行传送程序；

2. 中断请求在**指令执行结束后**检测，异常在**指令执行过程中**检测；

3. 磁盘的最小读写单位是**扇区**，不是字节。

4. CPU在**内核态（如执行操作系统程序）**也能检测和响应中断，并非仅在用户态。

5. 中断I/O的核心是“外设准备好数据后发中断请求”，外设准备数据的时间可长于中断处理时间（无需同步）

6. 中断结束信号（EOI）是**CPU向中断控制器发送**的，用于通知中断已处理完成；外部设备不会向CPU发送中断结束信号。

7. 异步串行通信的帧格式包含：1位起始位（不会提及）及其他

## 第六章 计算机的运算方法

### 一、机器数的编码表示

机器数是数值的二进制编码形式，常见编码及核心规则如下：

| 编码类型 | 正数表示（符号位+数值位）  | 负数表示                           | 真值0的表示形式                     |
| -------- | -------------------------- | ---------------------------------- | ----------------------------------- |
| 原码     | 符号位为0，数值位=真值本身 | 符号位为1，数值位=真值绝对值       | 2种（+0：`000...0`；-0：`100...0`） |
| 反码     | 符号位为0，数值位=真值本身 | 符号位为1，数值位=真值绝对值取反   | 2种（+0：`000...0`；-0：`111...1`） |
| 补码     | 符号位为0，数值位=真值本身 | 符号位为1，数值位=真值绝对值取反+1 | 1种（`000...0`）                    |
| 移码     | 补码的符号位取反           | 补码的符号位取反                   | 1种（补码0的符号位取反）            |

### 二、补码的核心特点（高频考点）

补码是计算机定点数的主流表示方式，核心特点：

1. **零的表示唯一**：区别于原码、反码；

2. **符号位参与运算**：符号位与数值位共同参与加减，无需单独处理；

3. **减法转加法**：减法可通过“减数取补后与被减数相加”实现；

   注：与真值的对应关系不如原码直观。

### 三、机器数的移位操作

移位分**算术移位（有符号数，以补码为例）和逻辑移位（无符号数）**：

| 移位类型 | 方向    | 规则（8位机器数）                         |
| -------- | ------- | ----------------------------------------- |
| 算术左移 | 左移1位 | 符号位不变，各位左移，末位补0（可能溢出） |
| 算术右移 | 右移1位 | 符号位补至高位，各位右移（保持符号）      |
| 逻辑左移 | 左移1位 | 高位丢弃，末位补0                         |
| 逻辑右移 | 右移1位 | 高位补0，低位丢弃                         |

### 四、补码的符号扩展

补码扩展位数时，规则由符号位决定：

- 正数（符号位为0）：扩展高位补0；

- 负数（符号位为1）：扩展高位补1。

1. `short` 和 `unsigned short` 均为 **16 位** 数据类型（固定位数，是推导核心）；
2. 计算机中，**有符号整数（short/int/long）一律以「补码」形式存储和解读**；

例：16位补码`0x8FA0`（符号位为1）扩展为32位，结果为`0xFFFF 8FA0`。

### 五、机器数与真值的转换

不同编码转真值的方法：

1. **原码/反码**：符号位为0→正数（取数值位）；符号位为1→负数（数值位取绝对值，反码需先取反）；

2. **补码**：符号位为0→正数（取数值位）；符号位为1→负数（**符号位不变，数值位「取反 + 1」得到绝对值，最终加负号**）；

3. **移码**：先转补码（符号位取反），再按补码规则转真值。

### 六、浮点数的编码结构

以IEEE 754标准为例，浮点数编码为“符号位+阶码+尾数”：

- 符号位：表示正负；

- 阶码：偏移表示（如单精度偏移127）；

- 尾数：规格化后**隐含最高位“1”**；

- 隐含部分：**基数（通常为2）**，不在机器数中显式存储。

### 七、无符号数与有符号数的转换

无符号数转有符号数（补码存储）：

1. 无符号数的二进制直接作为有符号数的补码；

2. 按补码规则转真值。

例：`unsigned short`的`65535`（二进制全1）转`short`，对应补码真值为-1。

## 第七章 指令系统

### 一、指令的基本格式

指令由**操作码**（表示指令功能）和**地址码**（表示操作数/操作数位置）组成，按地址码数量可分为：三地址指令、二地址指令、单地址指令、零地址指令。

### 二、操作码扩展技术（固定字长下扩展指令数）

为在固定指令字长中表示更多指令，采用“操作码扩展”技术（利用地址码位扩展操作码），核心步骤：

- **计算初始操作码位数**：

操作码位数 = 指令字长 - 地址码总位数（地址位数×**几**地址）；

- **计算初始操作码的最大指令数**：

最大指令数 =  2^操作码位数 ；

- **扩展操作码**：

剩余操作码数 = 最大指令数 - 已定义的指令数；

扩展后的指令数 = 剩余操作码数 ×  2^被扩展的地址码位数 （将一个地址码段作为操作码的扩展位）；

- **指令字长的确定**：

若按字节编址，指令字长需取**字节的整数倍**（如计算得23位，实际取24位）。

### 三、寻址方式（操作数的定位方法）

寻址方式是**确定操作数有效地址**的方法，核心分类及特点如下：

- **基础寻址方式（独立存在）**：立即寻址、寄存器寻址、内存寻址（直接寻址、间接寻址）
- **偏移寻址（统称类，包含3类具体寻址方式）**：  基址寻址、变址寻址、相对寻址
- **复合寻址**：  先变址后间址：由变址寻址和间接寻址组合而成



| 寻址方式   | 核心特点                                                     | 适用场景                                                     | 核心考点                                                     |
| :--------- | :----------------------------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| 立即寻址   | 操作数直接包含在指令中                                       | 需固定常数                                                   | 无需访存，**速度最快**                                       |
| 寄存器寻址 | 操作数存于寄存器，用寄存器编号表示位置，编号位数远少于内存地址位数 | 频繁访问的操作数                                             | 1. 可缩短地址段位数；2. 访存速度仅次于立即寻址               |
| 直接寻址   | 地址码为操作数的内存地址                                     | 访问固定内存单元                                             | 1.一次访存；2. 比间接寻址的速度快                            |
| 间接寻址   | 地址码为操作数地址的地址                                     | 灵活访问内存                                                 | 多次访存                                                     |
| 偏移寻址   | 通用规则：有效地址=寄存器内容+形式地址；包含基址、变址、相对寻址 | 需动态调整地址的各类场景                                     |                                                              |
| 基址寻址   | 有效地址=基址寄存器内容+形式地址；基址寄存器多存基址（如内存基址） | 内存空间分配、地址保护场景                                   | 1. 与变址寻址的区别（寄存器用途不同）                        |
| 变址寻址   | 有效地址=变址寄存器内容+形式地址；变址寄存器多固定（如存数组首址） | 按顺序访问一维数组（变址寄存器存数组首地址，形式地址存下标） | 1. 有效地址计算公式；2. 适合数组访问的原因；3. 操作数获取步骤 |
| 相对寻址   | 有效地址=当前PC值+相对位移量；PC为取完指令后的地址           | 转移指令                                                     | 1. 转移目标地址计算（关键算对PC值）；2. PC值的确定规则       |
| 复合寻址   | 先变址后间址                                                 | 需灵活定位且地址需二次解析的场景                             | 1. 寻址步骤（先变址再间址）；2. 最终有效地址的表示形式（((I)+D)） |

以“先变址后间址”为例，步骤：

1. 变址：计算中间地址 = 变址寄存器内容 + 形式地址；

2. 间址：将中间地址作为内存地址，访存得到**最终有效地址**（表示为  ((I)+D) ， I 为变址寄存器， D 为形式地址）.

### 四、寻址方式的计算

#### 1. 相对寻址（转移目标地址）

步骤：

1. 确定取完指令后的PC值： PC =指令所在地址+指令长度（字节数） ；

2. 目标地址 =  PC + 相对位移量 。

#### 2. 变址寻址（操作数获取）

步骤：

1. 计算有效地址： 有效地址= 变址寄存器内容+ 形式地址 ；

2. 获取操作数： 操作数= 有效地址对应的内存单元内容 。

## 第八章 CPU

### 一、CPU内部核心寄存器

CPU的核心寄存器用于存储指令、数据、状态等信息，各寄存器的功能与位数决定因素如下：

#### 1. 程序计数器（PC）

- **功能**：存储当前/下一条指令的内存地址，是指令执行顺序的核心控制部件。

- **位数决定因素**：由**存储器的容量（或地址位数）**决定；若指令按“字边界对齐”（如32位字对齐），地址的低n位（n为字占字节数的对数）恒为0，PC位数可扣除这些低位（例：4GB主存按32位字对齐，PC至少需30位）。

#### 2. 指令寄存器（IR）

- **功能**：存储当前正在执行的指令，供指令译码器解析。

- **位数决定因素**：由**指令字长**决定（需完整存放整条指令）。

#### 3. 通用寄存器组（GPRs）

- **功能**：暂存运算数据、操作数地址，是CPU内部数据交换的核心部件。

- **位数决定因素**：与**机器字长**一致（匹配数据的基本处理单位，保证运算效率）。

#### 4. 程序状态字寄存器（PSW）

- **功能**：表征程序和机器的运行状态，存储运算状态标志（如零标志、进位标志）、中断允许位等信息。

- **核心作用**：是**条件转移指令**的转移条件来源（依据PSW中的状态标志判断是否转移）。

### 二、指令周期中PC的变化

程序计数器（PC）在指令周期中会动态更新，核心规则：

#### 1. 取指操作后的PC内容

- 取指前：PC存储当前指令的地址；

- 取指过程中：PC自动自增（匹配指令的字节长度）；

- 取指完成后：PC存储**下一条指令的地址**。

#### 2. 无条件跳转指令的PC修改次数

在无条件跳转指令的指令周期中，PC会被修改2次：

1. 取指阶段：PC自增，指向“原顺序”的下一条指令地址；

2. 执行阶段：PC被修改为跳转目标地址。

### 三、间址周期的功能

间址周期是“间接寻址”对应的操作周期，核心作用是：

- 通过“形式地址”访问内存，获取**操作数的有效地址**；

- 间址周期结束时，内存数据寄存器（MDR）中存放的是“操作数的有效地址”（非操作数本身）。

### 四、指令执行涉及的部件

指令执行所需的部件由**寻址方式**和**操作类型**决定（以指令 `R[2] ← R[1] + M[R[0]]` 为例）：

1. **通用寄存器组（GPRs）**：提供源操作数（如 `R[1]`、`R[0]`）；

2. **算术逻辑单元（ALU）**：执行运算操作（如加法）；

3. **存储器（Memory）**：通过寄存器间接寻址（`M[R[0]]`）访存获取操作数。
